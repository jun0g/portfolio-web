# Personal Career Web Page

## Overview
이 프로젝트는 개인 커리어를 체계적으로 정리하고 공유하기 위한 웹 페이지이다.

기존 이력서 형식에서는 전달하기 어려운 프로젝트 경험, 문제 해결 과정, 기술 실험 기록 등을 보다 상세하게 정리하고 제공하는 것을 목표로 한다.

또한 구직 활동 시 이력서의 보조 자료로 활용하여, 프로젝트 경험과 기술적인 접근 방식, 문제 해결 과정을 보다 구체적으로 전달하기 위한 목적으로 제작되었다.

### Work Log
> 무중단 배포
- 노션 링크(https://www.notion.so/frosty-kr/2ea2da58b96948e6b7a940125152e9b4?source=copy_link)
- 문제 상황
    - EKS에 CI/CD 파이프라인 배포 환경에서 API 서버 배포 간 간헐적으로 서버 연결에 오류가 발생.
- 발생 원인
    - Pod의 종료 대기 시간이 애플리케이션의 종료 대기시간(graceful shudown) 보다 짧아서 애플리케이션이 완전히 종료되기 전에 Pod가 종료 되어 처리 중인 요청이 끊겨 발생한 오류.
- 해결 과정
    - CI/CD 파이프라인을 전체적으로 검토하고, Kubernetes에 설정된 Health Check 설정 확인.
    - 백엔드 개발자들과 이슈 공유 후, API 서버 설정된 graceful shutdown 설정 파악.
    - API 서버의 설정 보다 pod의 종료 대기 시간이 짧아 API 서버에서 처리 중인 요청이 pod 종료로 인해 끊김.
    - pod의 종료 대기 시간을 애플리케이션 보다 길게 설정하여 문제 해결.
- 적용 결과
    - 간헐적으로 발생하던 연결 오류 해결.

### Lab
접속 데이터를 활용한 악의적 사용자 분류 시스템 구현
- 실험 목적
    - 악의적 사용자 중 멀티 계정 유저를 클러스터링하여 유의미한 결과가 있는지 구현
- 실험 환경
    - Jupyter Notebook
- 실험 과정
    - 접속 데이터 시각화 및 데이터 전처리.
    - 클러스터링 수행 및 유의미한 클러스터 구분/분석.
- 결과 및 검증
    - 일부 유의미한 그룹을 도출함.
- 운영 적용 가능성
    - 운영 담장자 의견 - 접속 패턴으로 제제하기에는 제한이 있다. 클러스터별 IP 값이나 기존 오토 제재 캐릭터 유무 확인 필요.
    - 프록시, VPN을 통해 IP를 변죠해서 접근할 경우, 제재할 수 있는 근거가 부족함.
    - 개인 의견: 비정상적인 활동 로그를 가진 캐릭터를 제제하듯이 접속 패턴이 매우 유사한 캐릭터도 제제 가능해 보임.
    - 관련 판결 사례 존재.
- 한계 및 개선 방향
    - 비지도 학습의 한계.
    - 접속 패턴 샘플링 단위를 줄일 수록 정확도는 향상.
